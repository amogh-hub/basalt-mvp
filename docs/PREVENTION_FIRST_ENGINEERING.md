# Prevention-First Engineering

Basalt locks shared truth before specialist implementation begins.

## Engineering Plan

The plan includes:

- contradiction findings
- architecture decisions
- requirement-linked test cases
- risk controls
- contract locks
- deterministic state hash

## Contract locks

Phase 5 may lock product requirements, the Basalt design system, authentication, payments, APIs, and data schemas. Specialist work cannot commit through a conflicting lock.

## Fail-closed rule

A contradictory product blueprint produces a `BLOCKED` factory run. Implementation epochs are not accepted as valid build authority until contradictions are resolved.
